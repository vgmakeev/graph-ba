Ты — аудитор графа трассируемости. Задача — провести глобальный анализ всего графа и выявить проблемные артефакты, используя стратегию воронки: сначала быстрый структурный анализ, затем глубокий семантический ревью только подозрительных артефактов.

## Шаги

### 1. Убедись что граф актуален

Если есть сомнения — предложи `/reindex`.

### 2. Запусти глобальный аудит

```bash
uv run --with ~/dev/graph-ba graph-ba --json audit
```

Это вернёт JSON с тремя блоками:
- `summary` — общая статистика (артефакты, связи, проблемы, кандидаты)
- `issues` — все найденные проблемы (CYCLE, DANGLING, COVERAGE_GAP, MISSING_CROSS_LAYER, MISSING_BIDIR, BRIDGE, BOTTLENECK)
- `candidates` — приоритизированный список артефактов для ревью с причинами и приоритетом (high/medium)

### 3. Проанализируй результат аудита

Оцени масштаб проблем:
- Сколько всего кандидатов? Сколько HIGH-приоритета?
- Какие типы проблем доминируют?
- Есть ли системные паттерны (целый тип без покрытия, группа dangling)?

### 4. Запусти семантический ревью через параллельных субагентов

Отбери артефакты для глубокого ревью:
- Все кандидаты с `"priority": "high"` (обычно 5-15 штук)
- Кандидаты с `"priority": "medium"` — только если high < 10
- Максимум 15 артефактов суммарно

Раздели отобранных на группы по 3-5 штук. Для каждой группы запусти **отдельного субагента через Agent tool** одним сообщением (все субагенты параллельно):

Промпт каждому субагенту:

```
Ты — ревьюер артефактов бизнес-анализа. Проведи семантический ревью каждого артефакта из списка.

Артефакты для ревью: <ID-1>, <ID-2>, <ID-3>
Причины попадания в аудит: <reasons из candidates>

Для каждого артефакта:

1. Запусти команду:
   uv run --with ~/dev/graph-ba graph-ba review <ID> --semantic --lines 20

2. Прочитай исходный файл артефакта (путь виден в выводе review).

3. Проверь:
   - Полнота: все ли обязательные секции на месте?
   - Непротиворечивость: совпадают ли числовые значения (таймеры, лимиты, проценты) с указанными в связанных артефактах?
   - Трассируемость: все ли перекрёстные ссылки двусторонние? Нет ли dangling?
   - Специфичные проблемы из причин аудита (CYCLE, COVERAGE_GAP, MISSING_BIDIR и т.д.)

4. Верни краткий отчёт по каждому артефакту в формате:

## <ID> — <Название>
**Статус:** OK / WARN / FAIL
**Причины аудита:** <reasons>
**Найденные проблемы:**
- [КРИТИЧНО/ВАЖНО/НЕЗНАЧИТЕЛЬНО] описание
**Рекомендации:**
- что исправить
```

### 5. Собери результаты субагентов и сформируй итоговый отчёт

Дождись завершения всех субагентов. Агрегируй их находки:

```
## Глобальный аудит графа трассируемости

### Статистика
- Артефактов: N, связей: M
- Структурных проблем: K
- Глубокий ревью: X артефактов (Y субагентов)

### Критичные проблемы
1. [ID] описание — что делать

### Покрытие
- FEAT → REQ: XX% [GAP/WARN/OK]
  Не покрыты: ID-1, ID-2

### Системные паттерны
- Тип X систематически не имеет обратных ссылок на тип Y
- Кластер "Z" содержит dangling-ссылки

### Рекомендации по приоритету
1. [КРИТИЧНО] ...
2. [ВАЖНО] ...
3. [НЕЗНАЧИТЕЛЬНО] ...
```
