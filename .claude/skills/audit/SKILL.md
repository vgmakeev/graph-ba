Ты — аудитор графа трассируемости. Задача — провести глобальный анализ всего графа и выявить проблемные артефакты, используя стратегию воронки: сначала быстрый структурный анализ, затем глубокий семантический ревью только подозрительных артефактов.

## Шаги

### 1. Убедись что граф актуален

Если есть сомнения — предложи `/reindex`.

### 2. Запусти глобальный аудит

```bash
uv run --with ~/dev/graph-ba graph-ba --json audit
```

Это вернёт JSON с тремя блоками:
- `summary` — общая статистика (артефакты, связи, проблемы, кандидаты)
- `issues` — все найденные проблемы (CYCLE, DANGLING, COVERAGE_GAP, MISSING_CROSS_LAYER, MISSING_BIDIR, BRIDGE, BOTTLENECK)
- `candidates` — приоритизированный список артефактов для ревью с причинами

### 3. Проанализируй результат аудита

Оцени масштаб проблем:
- Сколько всего кандидатов? Сколько HIGH-приоритета?
- Какие типы проблем доминируют?
- Есть ли системные паттерны (целый тип без покрытия, группа dangling)?

### 4. Запусти семантический ревью для HIGH-приоритетных кандидатов

Используй **субагентов** для параллельного ревью. Каждый субагент получает 3-5 артефактов:

```
Для каждого артефакта из списка запусти:
uv run --with ~/dev/graph-ba graph-ba review <ID> --semantic --lines 20

Проверь:
- Полнота: все ли обязательные секции на месте?
- Непротиворечивость: совпадают ли числа в связанных артефактах?
- Трассируемость: все ли ссылки двусторонние?
Верни краткий отчёт по каждому артефакту.
```

Ограничения:
- HIGH-приоритет: ревью все (обычно 5-15 артефактов)
- MEDIUM-приоритет: ревью если HIGH < 10, иначе пропустить
- Максимум ~15 артефактов на глубокий ревью (контекстное окно)

### 5. Сформируй итоговый отчёт

```
## Глобальный аудит графа трассируемости

### Статистика
- Артефактов: N, связей: M
- Проблем найдено: K
- Глубокий ревью: X артефактов

### Критичные проблемы
1. ...

### Покрытие
- FEAT → REQ: XX% [GAP/WARN/OK]
- ...

### Системные паттерны
- ...

### Рекомендации
1. ...
```
