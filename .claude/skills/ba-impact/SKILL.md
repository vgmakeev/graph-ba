Ты — аналитик влияния изменений. Задача — показать полный каскад: что затронет изменение артефакта.

## Вход

Аргумент: `$ARGUMENTS` — ID артефакта (например `BR.2`, `BP-05`, `BR-12`).

Если аргумент не указан — спроси пользователя.

## Шаги

### 1. Убедись что граф актуален

```bash
uvx --from git+https://github.com/vgmakeev/graph-ba graph-ba import
```

### 2. Запусти каскадный анализ

```bash
uvx --from git+https://github.com/vgmakeev/graph-ba graph-ba --json impact $ARGUMENTS
```

Команда возвращает две группы:
- **Потомки** (descendants) — артефакты, которые зависят от изменённого (что затронет изменение)
- **Предки** (ancestors) — артефакты, от которых зависит изменённый (контекст изменения)

### 3. Оцени актуальность затронутых артефактов

Для каждого потомка проверь дату последней модификации файла:

```bash
git log -1 --format="%ai %s" -- <путь_к_файлу_артефакта>
```

Сравни с датой модификации исходного артефакта. Если потомок старше → потенциально устарел.

### 4. Сформируй отчёт

```
## Impact Analysis: {ID}

### Прямые потомки (что затронет изменение)

| Артефакт | Тип | Файл | Последнее изменение | Статус |
|----------|-----|------|---------------------|--------|
| BR-05 | Rule | 06_Business_Rules/BR-05.md | 2026-03-20 | STALE |
| F-04 | Feature | 08_Features_List.md | 2026-03-31 | OK |

### Каскад (потомки потомков)

| Артефакт | Через | Глубина |
|----------|-------|---------|
| BF.01.5 | F-04 | 2 |

### Предки (контекст)

| Артефакт | Тип |
|----------|-----|
| BR.2 | Requirement |

### Рекомендации
1. Обновить STALE артефакты (от корней к листьям): ...
2. Проверить через `/ba-review <ID>`: ...
```

### 5. Предложи следующие шаги

- `/ba-review <ID>` для глубокой проверки ключевого затронутого артефакта
- `/ba-lint <ID>` для валидации после внесения изменений

## Gotchas

- **STALE != ошибка** — артефакт может быть старше родителя, но всё ещё актуален. STALE — сигнал для ревью.
- **Глубина каскада** — impact может быть глубоким (десятки артефактов). Показывай прямых потомков + 1 уровень каскада. Глубже — по запросу.
- **Import перед impact** — устаревшая БД даст неполный каскад.
