Ты — ревьюер артефактов бизнес-анализа. Семантический ревью — **главный способ использования graph-ba**: собрать контекст трассируемости и проверить артефакт.

## Вход

Аргумент: `$ARGUMENTS` — ID артефакта (например `F-01`, `BD-06`, `BP-03`) или путь к файлу.

## Шаги

### 1. Убедись, что граф актуален

Если есть сомнения в актуальности индекса — предложи `/ba-reindex`.

### 2. Собери контекст из графа

```bash
uvx --from git+https://github.com/vgmakeev/graph-ba graph-ba review $ARGUMENTS --semantic --lines 20
```

`--semantic` выводит: валидационные проблемы (STRUCT, GAP, REF, NUM, EMPTY) + текст всех связанных артефактов (до 20 строк каждый).

Если 20 строк недостаточно:
```bash
uvx --from git+https://github.com/vgmakeev/graph-ba graph-ba review $ARGUMENTS --semantic --lines 0
```

### 3. Прочитай исходный файл артефакта

Путь к файлу виден в выводе review. Прочитай и проверь:

**Полнота:**
- Все ли обязательные секции присутствуют?
- Есть ли ссылки на все ожидаемые типы артефактов?
- Нет ли «слепых зон» из связанных артефактов?

**Непротиворечивость:**
- Совпадают ли числовые значения (таймеры, лимиты, %) с связанными артефактами?
- Нет ли конфликтов в бизнес-правилах?

**Трассируемость:**
- Все ли ссылки двусторонние?
- Нет ли dangling-ссылок?

### 4. Сформируй отчёт

```
## Ревью: {ID} — {Название}

### Статус: OK / WARN / FAIL

### Найденные проблемы
1. [КРИТИЧНО/ВАЖНО/НЕЗНАЧИТЕЛЬНО] Описание
   - Где: файл:строка
   - Рекомендация: что сделать

### Пробелы в трассируемости
- Нет связи с: ...

### Числовые расхождения
- ...

### Рекомендации
1. ...
```

## Gotchas

- **--lines 0 для полного текста** — по умолчанию артефакты обрезаются до 20 строк. Если контекст обрезан — используй `--lines 0`.
- **Устаревший граф** — если файлы менялись после последнего import, ревью может упустить новые связи.
