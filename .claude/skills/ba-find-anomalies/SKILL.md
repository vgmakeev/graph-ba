Ты — аналитик качества графа трассируемости. Задача — найти структурные аномалии и объяснить их значение.

## Шаги

### 1. Запусти детекцию аномалий

```bash
uvx --from git+https://github.com/vgmakeev/graph-ba graph-ba --json anomalies
```

### 2. Запусти матрицу покрытия

```bash
uvx --from git+https://github.com/vgmakeev/graph-ba graph-ba --json coverage
```

### 3. Проанализируй каждый тип аномалии

**Острова (ISLAND)** — группы, не связанные с основным графом.
- Пробел в документации или ожидаемо?

**Циклы (CYCLE)** — артефакты по кругу.
- Ожидаемые циклические зависимости или ошибка?

**Dangling (DANGLING)** — ссылки на несуществующие артефакты.
- Ещё не создан или ошибка в ID?

**Мосты (BRIDGE)** — единственные связи между частями графа.
- Насколько хрупко? Нужны ли доп. связи?

**Bottleneck-узлы** — слишком много связей.
- Ожидаемо для типа или перегрузка ответственностью?

**Sink-узлы** — без исходящих связей (тупики).
- Ожидаемо для листовых типов? Для BP/F — может быть недоработка.

### 4. Отчёт

```
## Анализ аномалий графа

### Критичные
- ...

### Требуют внимания
- ...

### Информационные
- ...

### Рекомендации по улучшению трассируемости
1. ...
```

## Gotchas

- **Острова — не всегда ошибка** — отдельные артефакты (например, архитектурные решения) могут быть изолированы намеренно.
- **Import первым** — если граф устаревший, аномалии будут ложными.
